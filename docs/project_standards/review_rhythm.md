# Review Rhythm

Review should happen in small batches while the context is still fresh.

## Default Rhythm

- batch work in pairs when possible
- review after each boundary is completed
- stop when a contract mismatch appears

## What To Review

- file responsibility
- stage input and output shape
- abstraction creep
- responsibility leakage
- fake verification
- mismatch between the claim and the proof

## Verification Standard

Verification must use a real check.

- run the code
- inspect one real artifact
- confirm counts or output shape

Do not accept "looks correct" as proof.
Do not accept a proof that demonstrates a different behavior than the one being claimed.
